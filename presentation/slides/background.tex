\begin{frame}[hoved]
\frametitle{Background: Timing Channels}

Three categories of timing channels in operating systems:

\vspace{0.3cm}
\begin{itemize}
  \item \textbf{Scheduler-based:} Priority decisions leak runnability info
  \item \textbf{Shared-state:} Caches, TLBs, branch predictors retain cross-task state
  \item \textbf{Delay-based:} Interrupts and locks delay execution predictably
\end{itemize}

\vspace{0.3cm}
\textbf{Isolation as the countermeasure:}
\begin{itemize}
  \item \textit{Spatial:} Memory partitioning (MMU / MPU)
  \item \textit{Temporal:} Ensuring shared resource usage in one domain does not affect another
\end{itemize}

\end{frame}

\begin{frame}[hoved]
\frametitle{The \texttt{fence.t} Instruction}

\begin{itemize}
  \item Hardware-assisted temporal isolation for RISC-V
  \item Flushes on-core microarchitectural state on domain switches:
    \begin{itemize}
      \item L1 caches, TLBs, branch predictors
      \item LFSR state, memory arbiters (full flush)
    \end{itemize}
  \item Overhead in seL4: only \textbf{0.7\%}
  \item Software-supported variant (\texttt{fence.t.s}) for out-of-order cores: \textbf{1.0\%}
\end{itemize}
