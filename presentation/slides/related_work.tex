\frametitle{The Missing OS Abstraction}
\textbf{seL4 Time Protection}: Introduced the notion of time protection at the OS level.

\begin{itemize}
  \item Built as a modification to the existing seL4 kernel
  \item Copies the kernel code into each security domain
  \item Flushes shared hardware that cannot be spatially partitioned
  \item Cache coloring to partition higher-level caches
\end{itemize}

\begin{frame}
\frametitle{S3K capability-based RTOS}
\textbf{S3K} provides a dynamic, capability-based approach.

\begin{itemize}
  \item A capability-based model that extends to time slices
  \item A time-driven, capability-based scheduler with systematic use of \texttt{fence.t}
  \item Predictable, constant-time system calls
  \item Scratchpad memory for kernel code execution (backed by L1 cache, flushed on scheduling)
\end{itemize}

\textbf{The Gap:} No prior work integrates these mechanisms into an \textit{existing} mainstream OS like FreeRTOS.
